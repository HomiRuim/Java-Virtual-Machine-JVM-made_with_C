◆ Frame

typedef struct frame Frame

Estrutura que representa um Frame de Execução.

O Frame contém:

  • Um ponteiro para o ClassFile da classe onde o método está sendo executado.
  • Um ponteiro para o MethodInfo do método em execução.
  • Um vetor de Slots para as Variáveis Locais.
  • Um vetor de Slots para a Pilha de Operandos.
  • Um ponteiro para o topo da Pilha de Operandos (para gerenciar o push/pop).
  • Um ponteiro para o próximo Frame na pilha de Frames (Call Stack).

O Frame terá um tamanho único e fixo (vetor de Slots), e os ponteiros (local_vars, operand_stack) serão usados para gerenciar o acesso dentro desse vetor.

◆ Slot

typedef u4 Slot

Tipo genérico para representar um valor na Pilha de Operandos ou Variáveis Locais.

O Frame terá um tamanho fixo, então usamos um tipo que pode acomodar qualquer valor de 4 ou 8 bytes (u4 ou u8).

Para simplificar, usaremos u4 para representar um slot de 32 bits. Valores de 64 bits (long, double) ocuparão 2 slots.
